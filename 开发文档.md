# PanisBoot 开发文档

## 项目简介

PanisBoot 是一款现代化的后台管理系统脚手架，基于 Spring Boot 3 框架开发。项目采用模块化设计，提供了完整的权限管理、系统监控、代码生成等功能，适用于企业级后台管理系统的快速开发。

## 模块说明

### 1. panis-boot-common

基础模块，包含通用的工具类、常量、异常定义等与业务无关的代码。

**主要功能：**

- 统一响应结果封装（Result）
- 业务异常定义（BizException）
- 常量定义（JWT常量、请求常量、缓存常量等）
- 通用工具类（加密、IP处理、字符串处理等）

### 2. panis-boot-infrastructure

基础设施模块，包含与基础设施相关的代码，如数据库访问、缓存、配置管理等。

**主要功能：**

- 数据库配置（Druid、MyBatis-Plus）
- Redis配置
- Swagger/Knife4j配置
- 全局异常处理
- 全局拦截器
- 分页支持
- Jackson配置
- 数据权限上下文管理

### 3. panis-boot-modules

业务模块，包含所有业务逻辑的实现。

#### 3.1 system 模块

系统管理模块，提供核心的系统管理功能。

**功能列表：**

- 用户管理（SysUser）
- 角色管理（SysRole）
- 权限管理（SysPermission）
- 菜单管理（SysMenu）
- 组织管理（SysOrgUnits）
- 岗位管理（SysPosition）
- 字典管理（SysDict、SysDictItem）
- 通知管理（SysNotice）
- 数据权限（SysDataScope）

**分层结构：**

- **Facade层**：门面层，对外提供统一的业务接口
- **Service层**：服务层，实现具体的业务逻辑
- **Repository层**：数据访问层，使用MyBatis-Plus进行数据库操作
- **Domain层**：领域对象层，包含Entity、VO、BO、DTO

#### 3.2 monitor 模块

监控管理模块，提供系统监控和日志管理功能。

**功能列表：**

- 系统监控（MonSystem）
- 缓存监控（MonCache）
- 文件管理（MonFile）
- 调度器管理（MonScheduler）
- 操作日志（MonLogsOperation）
- 登录日志（MonLogsLogin）
- 错误日志（MonLogsError）
- 调度日志（MonLogsScheduler）

**特色功能：**

- 操作日志切面（OperationLogAspect）
- 定时任务调度（基于Quartz）
- 系统信息监控（基于Oshi）

#### 3.3 tools 模块

工具模块，提供开发工具支持。

**功能列表：**

- 代码生成器（ToolGeneratorTable、ToolGeneratorTableColumn）
- 数据表管理（DataTable）

### 4. panis-boot-admin

后台管理模块，项目的启动模块。

**主要功能：**

- 提供RESTful API接口
- 集成所有业务模块
- 统一请求入口
- 启动类配置

## 开发规范

### 1. 分层架构

项目采用经典的分层架构：

```
Controller (控制器层)
    ↓
Facade (门面层)
    ↓
Service (服务层)
    ↓
Repository (数据访问层)
    ↓
Database (数据库)
```

**各层职责：**

- **Controller层**：接收HTTP请求，参数校验，调用Facade层，返回响应结果
- **Facade层**：门面层，对外提供统一的业务接口，协调多个Service
- **Service层**：业务逻辑层，实现具体的业务逻辑
- **Repository层**：数据访问层，使用MyBatis-Plus进行数据库操作

### 2. 命名规范

#### 2.1 包命名

- `controller`：控制器层
- `facade`：门面层
- `service`：服务层接口
- `service.impl`：服务层实现
- `repository`：数据访问层
- `repository.mapper`：Mapper接口
- `domain.entity`：数据库实体
- `domain.vo`：视图对象
- `domain.bo`：业务对象
- `domain.dto`：数据传输对象

#### 2.2 类命名

- **Controller**：`XxxController`
- **Facade**：`IXxxFacade`
- **Service**：`IXxxService`
- **ServiceImpl**：`XxxServiceImpl`
- **Mapper**：`XxxMapper`
- **Entity**：`Xxx`
- **VO**：`XxxVO`
- **BO**：`XxxBO`
- **DTO**：`XxxDTO`

#### 2.3 方法命名

- **查询**：`get`、`list`、`page`
- **新增**：`save`、`add`
- **修改**：`update`、`edit`
- **删除**：`remove`、`delete`
- **统计**：`count`、`statistics`

### 3. 代码规范

#### 3.1 Controller层

```java
@RestController
@Tag(name = "用户管理")
@RequiredArgsConstructor
@RequestMapping("/sys_user")
public class SysUserController {

    private ISysUserFacade sysUserFacade;

    @GetMapping("/page")
    @SaCheckPermission("sys:user:page")
    @Operation(summary = "获取用户管理列表")
```

- 使用`@SaCheckPermission`进行权限控制

- 使用`@Valid`进行参数校验
- 返回统一的`Result`对象

#### 3.3 Service层

```java
public interface ISysUserService extends IService<SysUser> {

    RPage<SysUserVO> listSysUserPage(PageQuery pageQuery, SysUserSearchDTO sysUserSearchDTO);

    SysUserVO get(Long id);
}

@Service
@RequiredArgsConstructor
public class SysUserServiceImpl extends ServiceImpl<SysUserMapper, SysUser> implements ISysUserService {
```

#### 1.4 创建Service

```java
public interface IDemoService extends IService<Demo> {

    RPage<DemoVO> listDemoPage(PageQuery pageQuery, DemoSearchDTO demoSearchDTO);

    DemoVO get(Long id);
}

@Service
@RequiredArgsConstructor
public class DemoServiceImpl extends ServiceImpl<DemoMapper, Demo> implements IDemoService {

    @NonNull
    private DemoMapper demoMapper;

    @Override
    public RPage<DemoVO> listDemoPage(PageQuery pageQuery, DemoSearchDTO demoSearchDTO) {
        Page<Demo> page = demoMapper.selectPage(pageQuery.toPage(), buildQueryWrapper(demoSearchDTO));
        return RPage.build(page, DemoVO.class);
    }

    @Override
    public DemoVO get(Long id) {
        Demo demo = demoMapper.selectById(id);
        return BeanUtil.copyProperties(demo, DemoVO.class);
    }

    private LambdaQueryWrapper<Demo> buildQueryWrapper(DemoSearchDTO demoSearchDTO) {
        return new LambdaQueryWrapper<Demo>()
                .like(StringUtils.isNotBlank(demoSearchDTO.getName()), Demo::getName, demoSearchDTO.getName())
                .eq(demoSearchDTO.getStatus() != null, Demo::getStatus, demoSearchDTO.getStatus())
                .orderByDesc(Demo::getCreateTime);
    }
}
```

#### 1.5 创建Facade

```java
public interface IDemoFacade {

    RPage<DemoVO> listDemoPage(PageQuery pageQuery, DemoSearchDTO demoSearchDTO);

    DemoVO get(Long id);

    Long save(DemoSaveDTO demoSaveDTO);

    Boolean update(DemoUpdateDTO demoUpdateDTO);

    Boolean remove(Long id);
}

@Component
@RequiredArgsConstructor
public class DemoFacadeImpl implements IDemoFacade {

    @NonNull
    private IDemoService demoService;

    @Override
    public RPage<DemoVO> listDemoPage(PageQuery pageQuery, DemoSearchDTO demoSearchDTO) {
        return demoService.listDemoPage(pageQuery, demoSearchDTO);
    }

    @Override
    public DemoVO get(Long id) {
        return demoService.get(id);
    }

    @Override
    public Long save(DemoSaveDTO demoSaveDTO) {
        Demo demo = BeanUtil.copyProperties(demoSaveDTO, Demo.class);
        demoService.save(demo);
        return demo.getId();
    }

    @Override
    public Boolean update(DemoUpdateDTO demoUpdateDTO) {
        Demo demo = BeanUtil.copyProperties(demoUpdateDTO, Demo.class);
        return demoService.updateById(demo);
    }

    @Override
    public Boolean remove(Long id) {
        return demoService.removeById(id);
    }
}
```

### 2. 权限管理

#### 2.1 权限注解

使用`@SaCheckPermission`注解进行权限控制：

```java
@SaCheckPermission("sys:user:page")
```

权限格式：`模块:功能:操作`

#### 2.2 权限配置

在数据库的`sys_permission`表中配置权限信息，包括：

- 权限标识（permission_code）
- 权限名称（permission_name）
- 权限类型（permission_type）
- 权限路径（permission_path）

#### 2.3 角色权限关联

在`sys_role_permission`表中关联角色和权限，一个角色可以拥有多个权限。

### 3. 数据权限

#### 3.1 数据权限注解

使用`@DataScope`注解进行数据权限控制：

```java
@DataScope(deptAlias = "d", userAlias = "u")
```

#### 3.2 数据权限类型

- **全部数据权限**：可以查看所有数据
- **自定义数据权限**：可以查看指定部门的数据
- **本部门数据权限**：只能查看本部门的数据
- **本部门及以下数据权限**：可以查看本部门及子部门的数据
- **仅本人数据权限**：只能查看自己的数据

#### 3.3 数据权限配置

在`sys_data_scope`表中配置数据权限信息，在`sys_role_data_scope`表中关联角色和数据权限。

### 4. 操作日志

#### 4.1 操作日志注解

使用`@OperationLog`注解记录操作日志：

```java
@OperationLog(title = "用户管理", businessType = BusinessType.INSERT)
```

#### 4.2 操作日志类型

- `INSERT`：新增

- `EXPORT`：导出

#### 4.3 操作日志记录

### 5. 异常处理

#### 5.1 业务异常

使用`BizException`抛出业务异常：

```java
throw new BizException("用户不存在");
throw new BizException(ResultCode.USER_NOT_EXIST);
```

#### 5.2 全局异常处理

全局异常处理器会统一处理所有异常，返回统一的错误响应：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BizException.class)
    public Result<Void> handleBizException(BizException e) {
        log.error("业务异常：{}", e.getMessage());
        return Result.error(e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常：", e);
        return Result.error("系统异常，请联系管理员");
    }
}
```

### 6. 缓存使用

```java
String value = redisUtil.get("key");
```

#### 6.2 缓存注解

使用Spring Cache注解：

```java
@Cacheable(value = "user", key = "#id")
public User get(Long id) {
    return userMapper.selectById(id);
}

@CacheEvict(value = "user", key = "#id")
public void delete(Long id) {
    userMapper.deleteById(id);
}

@CachePut(value = "user", key = "#user.id")
public User update(User user) {
    userMapper.updateById(user);
    return user;
}
```

```java
public Result<Void> importExcel(@RequestParam("file") MultipartFile file) {
    List<DemoDTO> list = ExcelUtil.importExcel(file.getInputStream(), DemoDTO.class);
    demoService.saveBatch(list);
    return Result.success();
}
```

## 常见问题

### 1. 启动失败

#### 问题：数据库连接失败

**解决方案：**

- 检查数据库是否启动
- 检查数据库连接信息是否正确
- 检查数据库用户权限

#### 问题：Redis连接失败

**解决方案：**

- 检查Redis是否启动
- 检查Redis连接信息是否正确
- 检查Redis密码是否正确

### 2. 权限问题

#### 问题：权限不生效

**解决方案：**

- 检查权限注解是否正确
- 检查权限配置是否正确
- 检查用户是否登录
- 检查角色权限关联是否正确

### 3. 数据权限问题

#### 问题：数据权限不生效

**解决方案：**

- 检查数据权限注解是否正确
- 检查数据权限配置是否正确
- 检查用户部门信息是否正确

### 4. 缓存问题

#### 问题：缓存不生效

**解决方案：**

- 检查Redis是否启动
- 检查缓存注解是否正确
- 检查缓存key是否正确
- 清除缓存重试
